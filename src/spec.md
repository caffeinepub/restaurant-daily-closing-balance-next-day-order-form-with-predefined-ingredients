# Specification

## Summary
**Goal:** Deploy the existing Restaurant Inventory app to production, verify frontend-to-backend connectivity in production, and provide the live app URL.

**Planned changes:**
- Trigger and complete a production deployment of the current app build (frontend + backend).
- Validate production connectivity so the frontend loads without backend connection errors.
- Verify in production that users can create/save a Daily Entry for a selected Balance Date and view it in History and Record Details.
- Add a repository-root `LIVE_URL.md` file containing the full https production URL for end-users.

**User-visible outcome:** Users can open the provided production link in a browser, use the app without connectivity errors, create a Daily Entry, and view it in History and Record Details.
